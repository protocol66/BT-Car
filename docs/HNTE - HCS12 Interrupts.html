HCS12 Interrupts

Interrupts are one of the most useful functions of the microcontroller, they are also a pain to set up. An interrupt does exactly what the name suggests, it interrupts the normal flow of code to run code in an interrupt service routine, then jumps back to the code that was running before the interrupt. An interrupt can be triggered by an external or internal trigger, such an an ATD reading or a timer overflow.

I had a lot of trouble with my mini dragon board the first time because no one solution I found on the Internet seemed to work, I eventually got them to work by mixing up a few different solutions. I am using the HC9S12 mini dragon board with serial monitor installed, this method should work with any of the 9S12Dx256 chips with serial monitor.

Vector Table

The first thing you need to setup an interrupt is an interrupt vector table, it basically points the code in the right direction when the interrupt is triggered. The one below is what I used for the Fuzzy Urn project so it already has three interrupt methods in there, you can use it yourself by deleting the three implemented interrupts and replacing them in the table with UnimplementedISR.

PRM File

The next step is to point the compiler.. I think.. to this vector table (I don't fully understand some of this I just know it works), There should be a .prm file somewhere in your project folder, mine is called HCS12_Serial_monitor.prm. You need to add the name of the vector table to the ENTRIES section, in this case its _vect, you can see mine below.

Finally comment out the line "VECTOR 0 _Startup" if it is there at the bottom of the prm file, you don't need this anymore as the startup vector is defined in the _vect table.

Interrupt Service Routine

Finlay you need an interrupt service routine for each of the implemented interrupts, here is my ISR for the ATD interrupt which is in my ATD.c file. Try to keep the ISR's nice and short so your code doesn't get caught inside an interrupt for too long, as you can see all mine does is store the data in the ATD data register and then reset the interrupt and ATD.
